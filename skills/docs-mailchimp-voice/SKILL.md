# Mailchimp Voice for Documentation

## Overview
Write public documentation like a knowledgeable teammate helping someone get unstuck—clear, human, and helpful.

## When to Use
**Use this skill when:**
- Writing or reviewing READMEs
- Creating public API documentation
- Writing tutorials and guides
- Drafting release notes or changelogs
- Reviewing docs that sound robotic or formal

**Don't use for:**
- Internal engineering notes (unless explicitly requested)
- Spec documents or RFCs
- Code comments

## Core Voice Principles

| Principle | Instead of |
|-----------|------------|
| Friendly, not casual | Arrogant or overly formal |
| Confident | Academic or tentative |
| Plainspoken | Jargon-heavy or robotic |
| Direct ("you") | Impersonal phrasing |

## Quick Reference

**Writing rules:**
- Active voice (avoid passive)
- Short sentences (15–20 words)
- Simple verbs: use, set, run, fix
- Explain jargon when needed
- Scannable sections with lists
- Inline code for commands/identifiers

**Tone by situation:**

| Situation | Approach | Example |
|-----------|----------|---------|
| Teaching | Patient, encouraging | "If this is your first time, that's fine—we'll walk through it." |
| Errors | Helpful, actionable | "This error usually means the key is missing. Add it to your config, then try again." |
| Success | Clear next step | "You're all set. Next, let's connect this to your CI pipeline." |

## Before/After Examples

**General writing:**
- ✅ "Add your API key once, and you're ready to send your first request."
- ❌ "API key initialization is required prior to request execution."

**Instructions:**
- ✅ "Run the command below to start the server."
- ❌ "Execute the following command in order to initialize the server process."

**Onboarding:**
- ✅ "Start by installing the package. If this is your first time, that's fine—we'll walk through it."
- ❌ "Install the package before proceeding with the remaining steps."

## Common Mistakes

| Mistake | Fix |
|---------|-----|
| Passive voice | Convert to active: "is required" → "you need" |
| Robotic tone | Add human touch: "Configuration completed" → "You're all set" |
| Unexplained jargon | Define or simplify terms |
| Long paragraphs | Break into 2-3 sentence chunks with lists |
| Formal academic style | Write like helpful teammate, not textbook |

## Review Checklist

When reviewing documentation:
- [ ] Flag robotic or overly formal language
- [ ] Suggest clearer phrasing for hard-to-scan text
- [ ] Recommend splitting long paragraphs
- [ ] Call out unexplained jargon
- [ ] Verify active voice throughout
- [ ] Check if it sounds like a helpful teammate (not a spec or textbook)
